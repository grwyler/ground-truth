# Codex Execution Rules

You are implementing the Pre-Development Readiness Platform.

Before coding:

1. Read all relevant docs in `/docs`.
2. Identify the smallest valuable task.
3. Explain the implementation plan briefly.
4. Modify only the files needed for the task.
5. Add or update tests.
6. Run verification commands.
7. Update `/docs/codex/done-log.md`.

Do not implement multiple roadmap milestones at once.

For every task, produce:

- Summary of changes
- Files changed
- Tests added/updated
- Verification commands run
- Remaining gaps
- Suggested next task

Prioritize the MVP milestones in this order:

1. Foundational Platform Setup
2. AI Draft Generation
3. Decision Objects + Traceability
4. Approval System + Versioning
5. Readiness Engine + Dashboard
6. Override System
7. Jira Integration
8. MVP Validation
